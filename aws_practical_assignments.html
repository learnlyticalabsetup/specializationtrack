Topic 1: Cloud Introduction, VPC Design, Transit Gateway, PrivateLink

üèóÔ∏è Practical Assignment 1: Multi-VPC Enterprise Network Architecture

Case Study

TechCorp is expanding from a single office to 5 global locations (US East, US West, Europe, Asia, Australia). They need a scalable network architecture that supports 3 environments (Production, Staging, Development) across all regions. The architecture must allow controlled communication between environments, centralized internet access, and secure connectivity to on-premises data centers.

Tasks to Complete

  • Design and implement a hub-and-spoke VPC architecture using Transit Gateway
  • Create VPCs for each environment in 2 regions (6 VPCs total)
  • Configure Transit Gateway with route tables for environment isolation
  • Implement centralized internet access through a shared services VPC
  • Set up VPC Flow Logs for network monitoring
  • Configure Route 53 private hosted zones for internal DNS resolution
  • Create security groups and NACLs following principle of least privilege
  • Document network topology and routing decisions

Assessment Criteria

  • Network segmentation and security (25%)
  • Routing configuration and optimization (25%)
  • Documentation and architecture diagrams (20%)
  • Cost optimization considerations (15%)
  • Scalability and maintenance (15%)

Deliverables

  • CloudFormation templates for infrastructure
  • Network architecture diagram
  • Security analysis report
  • Cost estimation document

üîó Practical Assignment 2: Hybrid Cloud Connectivity with Direct Connect

Case Study

FinanceSecure Corp operates critical trading applications that require consistent, low-latency connectivity between their on-premises trading floor and AWS. They need 99.99% availability, sub-10ms latency, and dedicated bandwidth of 10Gbps. The solution must include backup connectivity and support for multiple VPCs across different AWS accounts.

Tasks to Complete

  • Design Direct Connect architecture with redundancy
  • Implement Virtual Interfaces (VIFs) for multiple VPCs
  • Configure BGP routing with path preference
  • Set up backup Site-to-Site VPN connectivity
  • Implement PrivateLink for secure service access
  • Create monitoring and alerting for connection health
  • Test failover scenarios and document RTO/RPO
  • Implement cross-account connectivity using RAM

Assessment Criteria

  • High availability design and implementation (30%)
  • Performance optimization and latency management (25%)
  • Security and compliance considerations (20%)
  • Monitoring and troubleshooting capabilities (15%)
  • Cost efficiency and resource optimization (10%)

Deliverables

  • Network connectivity architecture diagram
  • BGP configuration scripts and documentation
  • Failover testing procedures and results
  • Monitoring dashboard and alerting setup
  • Performance benchmark report
